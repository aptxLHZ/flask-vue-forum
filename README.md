【3. 关于本地部署与调试的说明】

我们提供了完整的本地部署指南，以确保项目可以在您的环境中顺利运行。

**3.1 环境准备**
- Python 3.8+
- Node.js 16+
- PostgreSQL 数据库（已安装并正在运行）

**3.2 后端配置 (`/backend` 目录)**
1. 进入 `/backend` 目录，创建并激活 Python 虚拟环境 (`venv`)。
2. 运行 `pip install -r requirements.txt` 安装所有 Python 依赖。
3. **关键步骤：** 在 `/backend` 目录下，有一个名为 `env.txt` 的文件，这是 `.env` 文件的模板。请将其**重命名为 `.env`**，并根据您本地的 PostgreSQL 配置，修改其中的 `DATABASE_URL`，特别是用户名、密码和数据库名。
   (示例: `DATABASE_URL="postgresql://postgres:your_password@localhost/your_db_name"`)
4. 在 pgAdmin 中创建您在 `.env` 文件中指定的数据库。
5. 在 `/backend` 目录下，运行 `flask db upgrade` 来创建所有数据表。
6. 运行 `flask run` 启动后端服务。

**3.3 前端配置 (`/frontend` 目录)**
1. 进入 `/frontend` 目录。
2. 运行 `npm install` 安装所有 Node.js 依赖。
3. 运行 `npm run serve` 启动前端开发服务器。

**3.4 访问**
- 打开浏览器，访问 `http://localhost:8080` 即可开始使用。


【4. 其他细节说明】

- **虚拟环境 (`venv`)：** 我们遵循了 Python 开发的最佳实践，在 `/backend` 目录中使用了 `venv` 虚拟环境。为了减小提交文件的大小，我们没有将 `venv` 文件夹本身打包，您在本地运行时需要自行创建和激活。
- **Node 模块 (`node_modules`)：** 同理，前端的 `/frontend/node_modules` 文件夹也未包含在压缩包内，需要通过 `npm install` 命令在本地生成。
- **代码结构：** 项目采用前后端分离的目录结构，`backend` 和 `frontend` 文件夹分别包含了所有的后端和前端代码，结构清晰，便于查阅。
